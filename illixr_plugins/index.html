ILLIXR plugins

Default Plugins

  • offline_imu_cam: Reads IMU data and images from files on disk, emulating a real sensor on the headset (feeds the application input measurements with timing similar to an actual IMU).

  • ground_truth_slam: Reads the ground truth from the same dataset as the offline_imu_cam plugin. This data can be compared against the measurements from offline_imu_cam for accuracy. Timing information is taken from the offline_imu_cam measurements/data.

  • kimera_vio: Runs Kimera-VIO (upstream) on the input, and outputs the headset's pose. In practice, this publishes a fairly slow pose, so IMU integration and pose prediction is required to infer a fast pose.

  • gtsam_integrator: Integrates over all IMU samples since the last published SLAM pose to provide a fast pose every time a new IMU sample arrives using the GTSAM library (upstream).

  • pose_prediction: Uses the latest IMU value to predict a pose for a future point in time.

  • gldemo: Renders a static scene (into left and right eye buffers) given the pose from pose_prediction.

  • timewarp_gl: Asynchronous reprojection of the eye buffers.

  • debugview: Renders a frame for debug information.

  • audio: TODO. By default this is disabled, since it requires an NVIDIA GPU.

  • hologram: TODO. By default this is disabled, since it requires an NVIDIA GPU.

Below this point, we will use Switchboard terminology. Read the API documentation on Switchboard for more information.

  • In the above figure, rectangles are plugins, cylinders are topics (the graph is bipartitioned between these two gorups).

  • Solid arrows from plugins to topics represent publishing.

  • Solid arrows from topics to plugins represent synchronous reading. They take some action for every event which gets published on the topic.

  • Solid arrows from topics to plugins represent asynchronous reading. They need to know just the latest event on their topic.

  • Imagine the topic as a trough filing with events from its publisher, being drained by its synchronous readers (AKA subscribers), while asynchronous readres just skim from the top.

See Writing Your Plugin to extend ILLIXR.

Other Supported Plugins

ILLIXR supports additional plugins to replace some of the default plugins.

  • open_vins: This is an alternate SLAM (upstream) that uses a MSCKF (Multi-State Constrained Kalman Filter) to determine poses via camera/IMU.
  • rk4_integrator: Integrates over all IMU samples since the last published SLAM pose to provide a fast-pose everytime a new IMU sample arrives using RK4 integration.

See Building ILLIXR for more information on adding plugins to a config file.
